This hybrid theory-and-lab session, led by Dr. Yogesh and structured around lecture materials designed by Dr. Anand Jayaraman, bridges the theoretical mechanics of steering large language models (LLMs) with practical deployment using the Azure OpenAI Service. The session begins by demystifying LLM token prediction probabilities and exploring how configuration parameters (Temperature, Top P, Penalties) shape token generation. It introduces [Prompt Engineering and In-Context Learning](../concepts/prompt-engineering-context-learning.md) as a rigorous enterprise governance tool, highlighting API statelessness, the structure of API roles (`system`, `user`, `assistant`), and prompt strategies such as zero-shot, few-shot (multi-shot), and Chain-of-Thought prompting.

A significant portion of the session covers the shift from "vibe check" demonstrations to systematic [Model Evaluation and Validation](../concepts/model-evaluation-validation.md). The curriculum defines three levels of evaluation: traditional metrics (e.g., BLEU precision for translation and ROUGE recall for summarization) and their failure in capturing semantic synonyms; LLM-specific risks like factual inaccuracy, toxicity measurements (using ML classifiers trained on datasets like Jigsaw's Toxic Comment Classification), bias metrics (using demographic-attribute paired prompts), and hallucination rates; and enterprise business KPIs (e.g., ROI uplift, Cost-to-Serve, and Call Deflection Rate). To manage these live outputs at runtime, the session delves into runtime [Human-AI Collaboration and Guardrails](../concepts/human-ai-collaboration-guardrails.md), analyzing structural constraints (JSON parsing, Pydantic), content moderation, and vulnerabilities to natural language prompt injections (e.g., the historical Microsoft Copilot/Bing Chat "Sydney" leak) alongside their mitigation defenses.

In the hands-on lab, students utilized Neo Pro credentials to access Microsoft Azure and deployed a base model (GPT-4.1) within the Azure OpenAI Playground. The class performed experiments in persona-based system customization (contrasting a generalized prompt with a medical doctor persona context) and dynamic prompt variables (parameterizing tokens like `{{company_name}}` and `{{product}}`). They systematically tuned parameters including Temperature, Top P, Frequency Penalty, Presence Penalty, Max Completion Tokens, and Stop Sequences. Finally, addressing the reality that guardrails cannot solve knowledge gaps, students constructed an Assistant-based [Retrieval-Augmented Generation (RAG)](../concepts/retrieval-augmented-generation.md) pipeline by uploading local PDF files (grounded via CV text and publications) to an Azure vector store to conduct document-grounded QA. The session concluded with an enterprise architectural debate contrasting cloud-based closed-source models with local open-source deployments (e.g., Ollama, Qwen, Hugging Face hubs exposed via ngrok tunnels) to maintain data privacy in highly regulated industries like banking.

# Key Concepts

- **Token Generation & Probability Steering** — LLMs act as next-token predictors by computing probability distributions over potential tokens. Engineers steer outputs by modifying inference parameters such as Temperature and Top P (Nucleus Sampling) to manage randomness.
- **In-Context Learning Roles** — Defining explicit boundaries via API roles:
  - `system` — Master instructions governing tone, safety constraints, and personality (functioning as an enterprise governance tool).
  - `user` — End-user input query.
  - `assistant` — The response generated by the model.
  This role-based division is core to [Prompt Engineering and In-Context Learning](../concepts/prompt-engineering-context-learning.md).
- **Prompt Chaining & API Statelessness** — Unlike consumer chat interfaces, model API endpoints are inherently stateless. To maintain context across a conversation, developers must explicitly pass back the history of prior turns as structured messages.
- **Traditional Evaluation Metrics** — Overview of BLEU (precision-focused for translation) and ROUGE (recall-focused for summarization) metrics. The class evaluated their limitations, highlighting how they fall into the "Exact Match" trap and fail to evaluate semantic matches (e.g., penalizing valid synonyms). This is detailed under [Model Evaluation and Validation](../concepts/model-evaluation-validation.md).
- **LLM Toxicity & Bias Classifiers** — Quantifying safety risks utilizing ML classifiers (such as Logistic Regression or Transformers like Detoxify) trained on annotated human datasets like the Jigsaw Toxic Comment Classification corpus. Bias is audited using paired prompts that alter only demographic attributes. These align with [Bias, Fairness, and Alignment](../concepts/bias-fairness-alignment.md).
- **Enterprise-Level Metrics** — KPIs connecting performance to financial ROI, including Call Deflection Rate, Cost-per-query, throughput/latency, NPS/CSAT, and Cost-to-Serve.
- **Runtime Guardrails & Output Parsers** — Structuring and validating outputs in production using schemas (JSON, Pydantic, Guardrails.ai, or LangChain Output Parsers) to enforce business policies and reject or retry malformed completions. This is covered in [Human-AI Collaboration and Guardrails](../concepts/human-ai-collaboration-guardrails.md).
- **Prompt Injection Vulnerabilities** — A critical threat in [AI Security and Robustness](../concepts/ai-security-robustness.md) where malicious user inputs bypass system instructions (e.g., the historical "Sydney" codename leak). Defenses include strict role separation, input sanitization, and specialized runtime validators.
- **Hyperparameter Optimization**:
  - **Temperature** — Scales logits before applying the softmax function to control predictive randomness (lower values result in deterministic, high-probability selections).
  - **Top P (Nucleus Sampling)** — resticts candidate tokens to a subset whose cumulative probability reaches a designated threshold, defining the vocabulary set before Temperature makes the final token selection.
  - **Frequency Penalty** — Mathematically penalizes tokens based on how often they have already appeared, preventing word repetition.
  - **Presence Penalty** — Penalizes tokens based on whether they have appeared at all, preventing topic repetition.
  - **Stop Sequences** — Target strings that notify the model to halt further token generation.
- **Vector Stores & Assistant-Based RAG** — Uploading documents, chunking, and converting text into dense embeddings to execute semantic search. This provides factual context to the model to resolve knowledge gaps, utilizing [Retrieval-Augmented Generation (RAG)](../concepts/retrieval-augmented-generation.md) as a cost-effective alternative to [Transfer Learning and Fine-Tuning](../concepts/transfer-learning-fine-tuning.md).
- **Enterprise Open-Source Architecture** — Utilizing local execution frameworks (Ollama or Qwen models with public-private IP bindings via ngrok) to ensure data compliance within highly secure network boundaries, avoiding transmission of internal assets to closed-source endpoints.

# Topics Covered

- **Lab Setup & Cloud Authentication:** Logging into the Neo Pro environment, managing multi-factor authentication, and deploying Azure OpenAI resources in standard regions (East US).
- **Base Model Deployment:** Spinning up GPT-4.1 completions endpoints, setting context windows, and utilizing model routers to dynamically select the optimal LLM.
- **Hands-on Persona Customization:** Demonstrating semantic shifts in model answers before and after imposing persona restraints (such as medical doctor contexts).
- **Parameterization of Prompts:** Using curly bracket syntax (`{{variable_name}}`) to define variables in system instructions and updating their values within Azure.
- **Interactive Parameter Tuning:** Deep-diving into Temperature, Top P, completion limits, and penalties (frequency and presence), mapping mathematical models to output behaviors.
- **Vector Store Creation:** Implementing RAG using Azure's Assistant tool, creating an in-memory vector store, uploading a local PDF document, and executing contextual QA.
- **Open-Source Orchestration:** Discussing local model execution with Ollama, configuring public tunnels via ngrok, and addressing private network security considerations in banking.
- **Asset Deallocation:** Deleting deployed services via Azure's "All resources" tab to prevent credit depletion and stopping the Neo Pro lab.
